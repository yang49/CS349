  13. case class Transforms(unitSquareSize: Int, canvasMargin: Double, cacheMargin: Double) extends Product with Serializable

    We have three important groups of measurements going on.

    We have three important groups of measurements going on. This class helps us transform between them.

    1. The "policy plane", expressed in abstract units, to represent positions of candidates and voters. Candidates are typically between -0.25 and +1.25 -- but that's really 0 to 1, plus or minus the canvasMargin. Voters use the same units but may be outside that range, thanks to the Gaussian distribution. These units are typically named "pos" or "position".

    2. The canvas upon which positions are drawn. These are measured in integer pixels, which is passed in at unitSquareSize.

    3. The cache used to look up distances between positions. This is also integers (because it's stored in an array with integer indices). At the moment the units are the same as the canvas but with an extra margin around it to catch the positions that are outside those bounds.

    It's not clear to me that this is the correct decision to keep accuracy. On the other hand, if the unitSquareSize is small we've already given up accuracy.

    There is a graphic in eViz/docs/policyArea.graffle that hopefully sheds additional illumination on this.

    Everything is a square, so can use the same conversions for both x and y coordinates.
